## 2026-04-24 ŌĆö `OpenAiChatEvaluator` reasoning-mode escape hatch

### BUG-038: `OpenAiChatEvaluator` cannot disable reasoning on reasoning-mode models
- **File:** `syara/src/engine/llm_evaluator.rs`, `syara/src/config.rs`
- **Status:** fixed (2026-04-24)
- **Discovered during:** `llm_context_shield` Phase 10e integration tests
  against LMStudio reasoning-mode loadouts (`google/gemma-4-31b`,
  `qwen/qwen3.6-35b-a3b`, etc.).
- **Description:** Request body was a fixed `serde_json::json!({...})`
  literal containing only `model`, `messages`, `temperature`,
  `max_tokens`, `stream`. Reasoning-on models (the dominant 2026 local
  loadout) consumed the full 8192-token budget inside `reasoning_content`
  and emitted empty `choices[0].message.content` with
  `finish_reason: "length"`. `extract_openai_content` already detected
  the failure (BUG-035) but the caller had no way to *fix* it ŌĆö the
  OpenAI-compatible `reasoning_effort` knob wasn't reachable through the
  builder.
- **Fix:**
  1. Added `OpenAiChatEvaluator::DEFAULT_REASONING_EFFORT = "none"` and
     two new struct fields: `reasoning_effort: Option<String>`,
     `extra_body: Vec<(String, serde_json::Value)>`.
  2. Added builder setters `reasoning_effort(impl Into<String>)`,
     `disable_reasoning_effort()` (escape hatch for strict servers), and
     `extra_body(key, serde_json::Value)` (forward-compatible knob).
  3. Extracted body construction into `pub(crate) fn build_request_body`
     with explicit insertion order: explicit fields ŌåÆ `reasoning_effort`
     ŌåÆ `extra_body` (so callers can override anything via `extra_body`).
  4. `evaluate` now calls `build_request_body` instead of inlining the
     `json!` literal.
  5. `resolve_openai_env_defaults` reads `SYARA_LLM_REASONING_EFFORT`;
     empty string ŌåÆ call `disable_reasoning_effort()`, any other value ŌåÆ
     `reasoning_effort(value)`, unset ŌåÆ keep builder default. Honours
     `SYARA_LLM_NO_ENV=1`.
  6. `Registry::populate_defaults` plumbs the resolved value into the
     default `"openai-api-compatible"` evaluator.
- **Behaviour change:** out-of-the-box requests now include
  `reasoning_effort: "none"`. Permissive servers ignore unknown body
  keys; strict servers can opt out via builder or env.
- **Regression tests** (`syara/src/engine/llm_evaluator.rs::tests`):
  - `build_request_body_includes_default_reasoning_effort`
  - `build_request_body_with_extra_body_keys_are_present`
  - `build_request_body_extra_body_can_override_explicit`
  - `build_request_body_disable_reasoning_effort_omits_field`
  - Extended `openai_chat_builder_accepts_all_knobs` to chain the new
    setters.
- **Verified:** `cargo test -p syara-x --features llm` (123 tests
  passing); `cargo clippy --all-features -- -D warnings` clean.
- **Real-backend verification:** still pending ŌĆö invoke
  `cargo test -p syara-x --features llm -- --ignored --nocapture integration_real_openai_chat`
  against an LMStudio reasoning-mode model (e.g. `qwen/qwen3.6-35b-a3b`)
  to confirm pass-through end-to-end.
- **Caller plumbing follow-up:** downstream `llm_context_shield` should
  add `[syara] reasoning_effort = "none"` to `SyaraConfig` and an
  `LCS_LLM_REASONING_EFFORT` env var that calls
  `OpenAiChatEvaluatorBuilder::reasoning_effort(...)` /
  `disable_reasoning_effort()` accordingly.
